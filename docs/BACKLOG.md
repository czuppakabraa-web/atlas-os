# BACKLOG

Version: 1.0

Purpose

The backlog contains every approved idea that is not part of the current roadmap.

Items stay here until they are planned for a sprint.

Nothing should be forgotten.

Nothing should exist only inside conversations.

# Backlog Rules

A backlog item:

- has not started
- has no implementation
- has no sprint

When work begins, the item moves from BACKLOG to ROADMAP.

---

# Final Rule

Nothing should exist only in memory.

If an idea is accepted but not implemented,

it belongs here.
